Новая версия проекта на стеке Astro + NestJS
===========================================

Структура:

- `frontend/` — клиентская часть на Astro (SSR/SSG, UI, формы, публичные страницы и личный кабинет).
- `backend/` — серверная часть на NestJS (REST/GraphQL API, аутентификация, бизнес-логика, работа с БД и файлами).

На время миграции старая версия на Laravel продолжает жить рядом, чтобы можно было пошагово переносить функциональность и сравнивать поведение.

## Запуск проекта

### Бэкенд (NestJS)

```bash
cd backend && npm run start:dev
```

Бэкенд будет доступен на `http://localhost:4000`

### Фронтенд (Astro)

```bash
cd frontend && npm run dev
```

Фронтенд будет доступен на `http://localhost:4321`

### Одновременный запуск

Откройте два терминала и запустите команды выше в каждом из них.

## Настройка окружения

Перед запуском убедитесь, что созданы файлы `.env` в папках `frontend/` и `backend/` на основе `env.example`.

Для копирования ключей из Laravel `.env` используйте скрипт:
```bash
bash copy-env-keys.sh
```

## Публикация в продакшене

⚠️ **ВАЖНО:** Перед публикацией обязательно настройте переменные окружения для продакшена.

Подробная инструкция: [DEPLOYMENT.md](./DEPLOYMENT.md)

**Основные моменты:**
- Все URL должны быть настроены через переменные окружения
- `JWT_SECRET` должен быть изменен на случайную строку
- `CORS_ORIGINS` должен содержать только разрешенные домены
- `COOKIE_SECURE=true` для HTTPS
- Настройте реальные данные для БД, S3 и SMTP

## Безопасность

Проект включает несколько уровней защиты:

- ✅ **Rate Limiting** - защита от брутфорса и DDoS
- ✅ **Helmet.js** - HTTP security headers
- ✅ **Валидация входных данных** - class-validator для всех endpoints
- ✅ **Валидация переменных окружения** - проверка при старте
- ✅ **Структурированное логирование** - для отслеживания ошибок

Подробная документация: [SECURITY.md](./SECURITY.md)
